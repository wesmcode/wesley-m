# Accessibility audit — wesley-m.com

**Date:** 2026-05-22
**Scope:** Static site (`index.html`, `resume.html`, `blog.html`, `services.html`, `styleguide.html`) + Payload blog frontend (`apps/blog/src/app/(frontend)/**`, `RichText.tsx`, `editor/blocks/`)
**Excluded:** Payload admin, legacy `blog/*.html`, `about.html` (confirmed deleted)
**Target:** WCAG 2.2 AA
**Posture:** Audit only. No code changed. Findings include "why" so you can judge edge cases.

Two parallel passes were run: an a11y/mobile specialist (deep WCAG, RichText/Lexical-aware) and a site auditor (static-site HTML/CSS layer). This document merges them.

## At a glance

| Severity | Static site | Blog frontend | Total |
|---|---|---|---|
| **P0 — broken** | 4 | 1 | **5** |
| **P1 — serious** | 9 | 2 | **11** |
| **P2 — moderate** | 11 | 2 | **13** |
| **P3 — polish** | 4 | 2 | **6** |

**Three dominant themes:**

1. **Opacity-based color tokens fail contrast.** Every page uses `rgba(43,56,86,0.5)` and `rgba(255,255,255,0.5)` for metadata, section labels, and footer copy. On their stated backgrounds these compute to **2.78:1** and **4.19:1** respectively — both below the 4.5:1 normal-text threshold. Token-level fix unblocks ~7 findings at once.
2. **Dead navigation.** `about.html` and both resume PDFs were deleted but five links still target them, including the primary services CTA. Plus `styleguide.html` still loads the deleted `css/about.css`.
3. **Reduced-motion + landmark gaps.** Several hover/scroll animations bypass `prefers-reduced-motion`. Case study detail page has no `<main>` landmark. Multiple `<aside>` callouts create landmark noise.

---

## Top 10 to fix first (ordered by user impact)

1. **Restore or redirect `about.html`** — broken from nav + footer on every page, plus the services CTA.
2. **Fix the resume PDF download** — primary conversion endpoint, dead button (`resume.html:273`).
3. **Add `<main>` landmark** to `apps/blog/src/app/(frontend)/work/[slug]/page.tsx` — screen reader landmark nav is broken on every case study.
4. **Replace `rgba(43,56,86,0.5)` on white** with `#7A8299` (4.5:1) in `css/blog.css:4` and `apps/blog/src/app/(frontend)/styles.css:327`. Fixes byline, meta, section labels in one shot.
5. **Remove `aria-label="Featured post"`** from the splash `<a>` at `blog.html:64`. It overrides the article title; move the label to the wrapper `<article>` instead.
6. **Restore the deleted `css/about.css`** reference on `styleguide.html:11` — the styleguide currently renders unstyled components that other code copies from.
7. **Add a skip link** to `resume.html` (only page missing one) and `id="main-content"` on its `<main>`.
8. **Lift `.cap-col li` opacity** in `css/home.css:307-309` from 0.65 to 0.75. One line, fixes capability grid contrast.
9. **Fix footer contrast** — change `rgba(255,255,255,0.5)` to `~0.62` in `css/base.css:119-123`. Cascades to every page.
10. **Add `prefers-reduced-motion` rules** for `.process-row` (`css/home.css`) and the case-card hover transition (`apps/blog/src/app/(frontend)/styles.css`).

## P0 — Blocking

### P0-1 · `about.html` is deleted but linked from every page
**Where:** `index.html:70`, `index.html:315`, `blog.html:308`, `services.html:272`, `services.html:375`
**WCAG:** 2.4.4 Link Purpose, 1.3.1 Info and Relationships
**Why:** Every "About" link 404s. The `services.html:272` CTA targets `about.html#engage`, which is the documented engagement funnel entry — silently losing that link cuts a conversion path the rest of the page is built around. Screen reader users who navigate by link list encounter a dead entry on every page.
**Suggested change:** Restore `about.html` with the `#engage` anchor, or redirect the URL to `services.html#engage` and update all five hrefs. Pick one; pick fast.

### P0-2 · Resume PDF download is dead
**Where:** `resume.html:273` → `wesley_melo_resume_es_remote.pdf`
**WCAG:** 2.4.4 Link Purpose
**Why:** The download button is the resume page's primary action — recruiter on a deadline, hits 404, leaves. The button looks fine visually, so this won't surface in any non-functional review.
**Suggested change:** Restore both PDFs at the referenced paths, or update the `href`+`download` attributes to the correct files (and confirm the EN/ES split is still desired).

### P0-3 · Case study detail page has no `<main>` landmark
**Where:** `apps/blog/src/app/(frontend)/work/[slug]/page.tsx:122` — uses `<article id="main-content">`
**WCAG:** 2.4.1 Bypass Blocks, 1.3.6 Identify Purpose
**Why:** The skip link in `layout.tsx:25` points at `#main-content` but that ID lands on an `<article>`, not `<main>`. NVDA/JAWS landmark navigation (the keystroke screen reader users hit reflexively to skip chrome) finds nothing. The hero section sits *before* the article id, so even the landmark jump misses the page-level header context.
**Suggested change:**
```tsx
<section className="case-hero-band ...">...</section>
<main id="main-content">
  <article className="case-article">...</article>
  {siblings && <section className="case-related">...</section>}
  <section className="case-cta-wrap">...</section>
</main>
```

### P0-4 · `styleguide.html` references deleted `css/about.css`
**Where:** `styleguide.html:11`
**WCAG:** N/A directly, but reference-integrity failure that cascades into a11y decisions.
**Why:** Every component that lived in `about.css` (value cards, track cards, result cards, CTA strip, background timeline) renders unstyled inside the styleguide. The styleguide is the documented source of truth (`docs/components.md` + `styleguide.html`); developers copying markup from a broken render will copy degraded patterns into production. Focus-style decisions made while looking at the styleguide will be made against a stripped baseline.
**Suggested change:** Either restore `css/about.css` or remove the link and fold any still-needed component styles into `css/styleguide.css`/`css/base.css`. Pick based on whether those components are still in use on `services.html` and elsewhere.

### P0-5 · `resume.html` has no skip link
**Where:** `resume.html` (no skip link anywhere; `<main>` at line 270 also has no `id`)
**WCAG:** 2.4.1 Bypass Blocks
**Why:** Every other page has a skip link. Inconsistency erodes the AT user's trust that the site is reliably navigable. Keyboard users on the resume page must tab through the download button and name block to reach content.
**Suggested change:** Add `<a href="#main-content" class="skip-link">Skip to resume</a>` and `id="main-content"` on `<main>` at line 270. Mirror the pattern used in `index.html`.

## P1 — Serious

### P1-1 · `rgba(43,56,86,0.5)` on white fails contrast (2.78:1)
**Where:** `css/blog.css:4` (token `--fg-hover`), applied to `.byline`, `.opinion-meta`, `.most-read-meta`, `.card-b-meta`, `.side-meta`, `.ad-slot-label`, `.reading-link-source`. Same pattern in `apps/blog/src/app/(frontend)/styles.css:325-328` (`.card-section` labels rendered by `page.tsx:51,86-89`).
**WCAG:** 1.4.3 Contrast Minimum (needs 4.5:1 for <18pt normal / <14pt bold; these are 10–12px uppercase)
**Why:** Metadata is how readers decide whether to click an article. Date, read time, category — at 2.78:1 in daylight on a phone these vanish. The primary impact group is **undiagnosed low-vision users** (~8% of people over 50) who don't identify as having a disability — they just stop reading and you never hear about it.
**Suggested change:** Replace `--fg-hover: rgba(43,56,86,0.5)` with a computed hex `#7A8299` (4.5:1 exactly). Token-level fix in `css/blog.css:4` and `styles.css:327`.

### P1-2 · `rgba(43,56,86,0.65)` for `.cap-col li` fails (4.12:1)
**Where:** `css/home.css:307-309`
**WCAG:** 1.4.3 Contrast Minimum
**Why:** The capabilities grid is substantive copy — a buyer scanning service items to evaluate fit. 14px regular weight, fails by 0.4:1.
**Suggested change:** Raise alpha to 0.75 (~4.70:1), or use `--fg-quiet`.

### P1-3 · Process section meta fails contrast on navy (4.19:1)
**Where:** `css/home.css:232` — `.process-meta`, 22px regular on `#2B3856`
**WCAG:** 1.4.3 Contrast Minimum
**Why:** Process meta is the only explanatory copy in the section. "Discover," "Shape," "Create," "Scale" are meaningless without the sentence beside them. At 4.19:1 it fails for any user with reduced contrast sensitivity.
**Suggested change:** Raise the white alpha from 0.5 to ~0.62 in this context (or use a computed hex).

### P1-4 · Multiple small labels on navy fail (4.19:1)
**Where:** `css/services.css:36-42, 202-206, 267-270` — `.stat-label`, `.section-label`, `.offer-group-label`, `.offer-meta`
**WCAG:** 1.4.3 Contrast Minimum
**Why:** Stat labels caption the three proof points ("8 production AI agents," etc.). Without legible labels the numbers lose context. Offer meta is duration/price under each service card — direct purchase signal.
**Suggested change:** Same token bump, 0.5 → 0.62 alpha on white-over-navy, or use `--fg-quiet` for ~6.90:1.

### P1-5 · Footer copyright fails contrast (4.19:1)
**Where:** `css/base.css:119-123` (`.footer-meta`), inherited by every page
**WCAG:** 1.4.3 Contrast Minimum
**Why:** Small uppercase (~12px), fails AA by a thin margin. One token change unblocks every page footer.
**Suggested change:** `.footer-meta { color: rgba(255,255,255,0.62); }`.

### P1-6 · `aria-label="Featured post"` swallows the article title
**Where:** `blog.html:64`
**WCAG:** 4.1.2 Name, Role, Value; 2.4.6 Headings and Labels
**Why:** Per the accessible-name spec, `aria-label` on `<a>` overrides all descendant text including the `<h3>` title. Screen reader users hear "Featured post, link" and have no idea what they'd be opening. The Payload version (`page.tsx:46`) gets this right — the label belongs on the wrapping `<article>`, not the link.
**Suggested change:** Delete the `aria-label` from `<a class="splash">`. Move it (or `aria-labelledby` pointing at the h3) onto the parent `<article>`/`<section>` if you want the "featured" semantic.

### P1-7 · `--fg-quiet` on cream just barely fails (4.45:1)
**Where:** `css/blog.css:5` (token), applied to `.splash-dek`, `.side-dek`, `.card-b-dek`, opinion items over `.opinion-band` (cream `#F4F2EC`)
**WCAG:** 1.4.3 Contrast Minimum
**Why:** Dek copy is the second-most-important reading signal after the title. Failing by 0.05:1 sounds trivial but cream backgrounds render slightly differently across engines, so the real-world ratio drifts below the threshold on some displays.
**Suggested change:** Set dek text explicitly to `rgba(43,56,86,0.75)` (4.70:1) rather than touching the global token, since `--fg-quiet` passes easily on white/navy.

### P1-8 · Carousel announcement gaps
**Where:** `index.html:124-174`, `js/carousel.js`
**WCAG:** 4.1.2 Name, Role, Value; 4.1.3 Status Messages
**Why:** Container has `role="list"` + `tabindex="0"` + arrow-key navigation. AT announces "What I bring, list, 7 items" and then accepts focus silently — no indication arrow keys do anything, no position announcement, no live region when auto-advance fires. Also: this is the social-proof component (7 competency cards) — high-value content for AT users to actually be able to read.
**Suggested change:** Add `aria-roledescription="carousel"` to the container. Add a polite `aria-live` region updated by `setActive(i)` with "Item N of 7: {card title}". Make cards individually focusable if the user is expected to interact with them.

### P1-9 · Mobile-only icon glyphs are read by screen readers
**Where:** `index.html:100-103` and similar — `<span class="mobile-only">[→︎]</span>`, `[↗︎]`, `[+]`
**WCAG:** 1.1.1 Non-text Content
**Why:** Visible decorative arrows are included in the accessible name. Mobile SR users hear "ABOUT right arrow" / "EMAIL plus sign" — noise on every link.
**Suggested change:** Add `aria-hidden="true"` to those spans. Visual users still see the glyph; AT skips it.

### P1-10 · `<main>` on `resume.html` has no landmark id
**Where:** `resume.html:270`
**WCAG:** 2.4.1 Bypass Blocks (paired with P0-5)
**Why:** Even after P0-5 lands, the target id needs to exist for the skip link to resolve.
**Suggested change:** Add `id="main-content"` on `<main>`. Same fix that satisfies the new skip link.

### P1-11 · Embedded iframes have generic title `"Embedded content"`
**Where:** `apps/blog/src/components/RichText.tsx:164`
**WCAG:** 4.1.2 Name, Role, Value
**Why:** Screen readers announce iframe titles before entering them. A YouTube video, a Spotify embed, and a Twitter quote are all described identically — AT users can't decide whether to enter the frame.
**Suggested change:**
```tsx
title={f.caption || `${info.kind} embed`}
```
That gives "youtube embed" / "spotify embed" / "twitter embed" automatically. Captions, where present, take precedence.

## P2 — Moderate

### P2-1 · `.card-section` token contrast on white (2.77:1)
**Where:** `apps/blog/src/app/(frontend)/styles.css:325-328` rendered in `page.tsx:51`, `86-89`
Same root cause as P1-1; flagged separately because it's in the blog frontend file rather than `css/blog.css`. Same fix.

### P2-2 · `prefers-reduced-motion` misses `.process-row`
**Where:** `css/home.css:202` (`transition`), `css/home.css:362-373` (existing reduced-motion block omits it)
**WCAG:** 2.3.3 Animation from Interactions
**Why:** Four large rows flash white on hover. Users with vestibular conditions who set the OS preference still get the motion. The block already exists — just missing one selector.
**Suggested change:** Inside the existing `@media (prefers-reduced-motion: reduce)`: `.process-row { transition: none; }`.

### P2-3 · `prefers-reduced-motion` missing entirely in blog frontend CSS
**Where:** `apps/blog/src/app/(frontend)/styles.css` — case-card hover (line 574-582), CTA button transitions
**WCAG:** 2.3.3 Animation from Interactions
**Why:** `/work` index page lifts each card on hover. Multiple cards = compounding motion. No reduced-motion block exists in the file at all.
**Suggested change:** Add a `@media (prefers-reduced-motion: reduce)` block covering `.case-card`, `.case-card:hover`, `.case-cta-btn`.

### P2-4 · Em-dashes/en-dashes in titles, OG tags, and body copy
**Where:** Across `index.html`, `services.html`, `blog.html`, `resume.html`, `styleguide.html` — including `<title>`, `og:title`, `meta name="description"`, JSON-LD `name`, and ~15 visible em-dashes in `services.html` body (lines 61, 69, 75, 98, 115, 163, 215, 237, 281, 292, 312, 332, 342).
**WCAG:** 3.1.5 Reading Level (cognitive)
**Why:** Violates the explicit CLAUDE.md project rule. AT announcement varies (NVDA: "em dash", JAWS: pause, VoiceOver: variable) — choppy TTS interrupts reading flow. Services page is doing selling work; choppy AT reading reduces conversion from AT users specifically.
**Suggested change:** Bulk replace per CLAUDE.md: `—` → period/colon, `–` in ranges → "to". The rule says grandfathered until next edit — this audit *is* the next edit prompt.

### P2-5 · `aria-label="Breadcrumb"` with non-breadcrumb markup
**Where:** `blog.html:38`, `services.html:52`
**WCAG:** 4.1.2 Name, Role, Value
**Why:** WAI-ARIA breadcrumb pattern requires `<ol>` with `<li aria-current="page">`. Both pages have a single bare `<a>` inside the labeled `<nav>`. AT announces "Breadcrumb navigation" then encounters one item with no current-page marker — the mental model immediately breaks.
**Suggested change:** Either rename the landmark (`aria-label="Back"`) or upgrade to proper breadcrumb markup with `<ol>` + `aria-current="page"`.

### P2-6 · `user-select: none` on content-bearing containers
**Where:** `css/home.css:35, 55` — `.row`, `.timeline-row`
**WCAG:** 1.3.1 Info and Relationships (loosely)
**Why:** These containers hold the h1, company names, date ranges. Blocking selection prevents sighted users from copying names/dates and can interfere with some AT selection-mode features (e.g., VoiceOver's "select text" rotor).
**Suggested change:** Remove `user-select: none` or restrict it to decorative children only.

### P2-7 · `target="_blank"` links missing `rel` and SR notification
**Where:** `services.html:378` (Resume link in footer, no `rel`), other new-tab links across pages with no AT cue
**WCAG:** 3.2.5 Change on Request
**Why:** Security: missing `rel="noopener noreferrer"` is a tab-hijack vector. UX: AT users get no warning the link opens elsewhere.
**Suggested change:** Add `rel="noopener noreferrer"` and either visually-hidden `(opens in new tab)` text or an aria-described pattern.

### P2-8 · 1px focus outlines on some blog cards
**Where:** `css/blog.css:118, 311, 429` — `.zone-link`, `.opinion-item`, `.most-read-item`
**WCAG:** 2.4.11 Focus Not Obscured / 2.4.13 Focus Appearance
**Why:** Other cards on the same page (splash, side-item, card-b, reading-link) use 2px outlines. Within one stylesheet the focus indicator is inconsistent — keyboard users get a degraded indicator on some cards only.
**Suggested change:** Standardize to 2px (`outline: 2px solid var(--fg)` with offset).

### P2-9 · Snap-section on `#site-footer` on home
**Where:** `index.html` — footer inherits `.snap-section`
**WCAG:** 2.4.3 Focus Order (loosely)
**Why:** Keyboard focus entering the footer can trigger a mandatory snap, jolting the scroll position mid-tab. Users tabbing through footer links experience unexpected viewport jumps.
**Suggested change:** Override `scroll-snap-stop` for `#site-footer` or move it outside the snap container.

### P2-10 · `index.html` has no `<h2>` headings
**Where:** `index.html` — jumps straight from `<h1>` to multiple `<h3>`s
**WCAG:** 2.4.10 Section Headings, 1.3.1 Info and Relationships
**Why:** AT users navigate by heading level — "go to next h2" is one of the most-used screen reader shortcuts. With no h2s, About / Process / Services sections can't be jumped to. Heading-level navigation is broken on the most important page on the site.
**Suggested change:** Introduce one `<h2>` per major section (About, Process, Services). Demote where needed.

### P2-11 · Carousel auto-advance with no pause control (Level **A**)
**Where:** `index.html:124-174`, `js/carousel.js`
**WCAG:** 2.2.2 Pause, Stop, Hide — Level A (the more serious tier)
**Why:** Auto-advance every 5s. SC 2.2.2 requires a user mechanism to pause/stop/hide moving content lasting >5s. Pausing on `focusin` doesn't satisfy the criterion — a user who hasn't focused the component has no way to stop the motion. This is the only Level A violation in the audit and the most legally exposed item.
**Suggested change:** Add a visible pause button. Or remove auto-advance (manual swipe/arrow only). The visible-button approach is faster and matches the W3C carousel pattern.

### P2-12 · Decorative typographic chars read by AT
**Where:** `index.html:50` — `.years` spans with `→︎ · · · ·`
**WCAG:** 1.1.1 Non-text Content
**Why:** Variation selector should suppress emoji presentation but doesn't reliably suppress AT announcement. Some AT reads "right arrow variation selector 15 middle dot middle dot…".
**Suggested change:** Wrap decorative glyphs in `<span aria-hidden="true">`.

### P2-13 · Empty `<figcaption>` rendered when no caption/credit
**Where:** `apps/blog/src/components/RichText.tsx:120-123`
**WCAG:** 1.3.3 Sensory Characteristics (AT noise)
**Why:** Every uploaded image gets a `<figcaption>` even when both caption and credit are empty. Some screen readers announce empty elements.
**Suggested change:** Conditional render — only emit the `<figcaption>` when caption or credit is present.

## P3 — Polish

### P3-1 · Email reveal not announced to screen readers
**Where:** `js/email.js:6-20`, used in `index.html:100`, `services.html:362,379`
**WCAG:** 4.1.3 Status Messages
**Why:** Activating "Email" replaces the link text with the address. AT not notified — user can't tell anything happened without re-focusing the element.
**Suggested change:** Add a visually-hidden `<span role="status" aria-live="polite">` to the layout; on reveal, set its text to the revealed address (or update the link's `aria-label` to the address).

### P3-2 · Verbose side-stack link accessible names
**Where:** `blog.html` side-stack links
**Why:** Computed name = kicker + title + dek + meta. Add `aria-hidden="true"` on kicker/dek/meta inside the link wrapper so the title alone names the link.

### P3-3 · Duplicate `<h1>` on `styleguide.html`
**Where:** `styleguide.html` — page h1 + demo Hero h1
**Why:** Multiple h1s confuse AT document outline. Demo headings should be `aria-hidden` or demoted to h3 inside an `<aside>`.

### P3-4 · `<aside>` overuse for callouts
**Where:** `apps/blog/src/components/RichText.tsx:144`
**Why:** `<aside>` is a landmark — multiple callouts per article = multiple unlabeled "complementary" landmarks.
**Suggested change:** Render callouts as `<div role="note" aria-label={f.title || 'Note'}>`.

### P3-5 · `.case-card-title::before { content: '— '; }`
**Where:** `apps/blog/src/app/(frontend)/styles.css:631-633`
**Why:** Pseudo-element injects an em-dash into the SR-read accessible name. Same project policy and AT-noise concern as P2-4.
**Suggested change:** Use a CSS border or a real `<span aria-hidden>` separator.

### P3-6 · Resume email link ambiguous initial state
**Where:** `resume.html:278` — `<a id="resume-email" href="#">email</a>`
**Why:** Lowercase "email" is a weak accessible name and `href="#"` jumps to top if the inline script hasn't run yet. Other email reveals on the site use the class-based pattern from `js/email.js` for consistency.
**Suggested change:** Migrate to `js/email.js`'s `.js-email` pattern; set initial accessible name to "Reveal email address".
